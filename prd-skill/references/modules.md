# PRD 可复用模块

常用模块，可直接插入 PRD 对应章节。

---

## 1. 用户故事卡片

**格式**
```
作为 [用户角色]，
我希望 [完成什么操作/获得什么功能]，
以便 [实现什么价值/解决什么问题]。

验收标准（Given-When-Then）：
- Given [前置条件]
- When [用户操作]
- Then [系统行为/预期结果]
```

**示例**
```
作为一个初次使用 App 的新用户，
我希望通过手机号快速完成注册，
以便无需记忆复杂账号即可开始使用服务。

验收标准：
- Given 用户打开注册页面
- When 用户输入有效手机号并点击「获取验证码」
- Then 系统在 30 秒内发送 6 位短信验证码，页面显示倒计时

- Given 用户输入正确的验证码
- When 点击「完成注册」
- Then 系统创建账号并自动登录，跳转至首页
```

## 5. 状态流转图（文字版）

```markdown
## [对象名称] 状态流转

**状态列表**：草稿 → 审核中 → 已发布 → 已下线 → 已删除

**流转规则**：

| 当前状态 | 触发操作 | 目标状态 | 执行人 | 条件 |
|----------|----------|----------|--------|------|
| 草稿 | 提交审核 | 审核中 | 作者 | 必填字段完整 |
| 审核中 | 审核通过 | 已发布 | 审核员 | - |
| 审核中 | 审核拒绝 | 草稿 | 审核员 | 需填写拒绝原因 |
| 已发布 | 手动下线 | 已下线 | 作者/管理员 | - |
| 已下线 | 重新发布 | 已发布 | 作者 | - |
| 任意状态 | 彻底删除 | 已删除 | 管理员 | 不可恢复 |
```
