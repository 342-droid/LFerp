# 入库业务流程图

基于《需求说明-入库》文档整理的 Mermaid 格式流程图，可在支持 Mermaid 的 Markdown 预览或 [Mermaid Live Editor](https://mermaid.live) 中查看。

## 1. 入库业务总览

```mermaid
flowchart TB
    subgraph 业务场景
        A1[采购入库]
        A2[仓配入库]
        A3[返配入库]
        A4[调拨入库]
        A5[退货入库]
    end

    subgraph 系统用例
        B1[UC01 生成入库单]
        B2[UC02 收货录入]
        B3[UC03 取消收货]
        B4[UC04 关闭入库单]
    end

    A1 & A2 & A3 & A4 & A5 --> B1
    B1 --> B2
    B2 --> B3
    B2 --> B4
```

## 2. UC01 生成入库单

```mermaid
flowchart TB
    Start([接口下发/上游通知]) --> A[根据订单信息匹配仓库]
    A --> B{匹配到仓库-货主?}
    B -->|否| B1[终止并写日志: 无匹配仓库-货主]
    B1 --> B2[配置仓库-货主后重新触发]
    B -->|是| C[根据单据类型/业务类型映射WMS入库类型]
    C --> D[查验 ibd_receipt 来源单号唯一]
    D --> E{来源单号唯一?}
    E -->|否| E1[拦截并写日志: 入库单号+来源单号不唯一]
    E -->|是| F[新增入库单 ibd_receipt 状态=初始]
    F --> G[新增入库单明细 ibd_receipt_detail 按商品汇总计划入库量]
    G --> H[用户可于 入库管理-入库单 查看]
    H --> End([结束])
```

## 3. UC02 收货录入（按商品收货）

```mermaid
flowchart TB
    Start([收货员进入收货菜单]) --> A[查询入库单: 扫一扫/按类型查询]
    A --> B{扫一扫输入单号}
    B --> C[系统查验 仓库+货主+单号/来源单号/关联单号]
    C --> D{匹配结果?}
    D -->|无记录| D0[提示/列表空]
    D -->|多条| D2[进入查询入库单列表]
    D2 --> D3[收货员选择入库单]
    D3 --> E{唯一且状态 in 初始/收货中 且 收货方式=商品收货?}
    D -->|唯一| E

    E -->|否| E1[不进入录入或提示]
    E -->|是| F[进入 入库单录入-扫描商品]
    F --> G[扫描/输入商品]
    G --> H{商品有效校验}
    H -->|无效| H1[提示: 商品无效]
    H -->|无此商品明细| H2[提示: 入库单无此商品]
    H -->|已收满| H3[提示: 此商品已完成收货]
    H -->|有效且在明细且未收满| I{是否效期管理?}
    I -->|是| I1[显示生产日期输入框]
    I -->|否| I2[生产日期默认 99999999]
    I1 --> J[进入 收货数量录入]
    I2 --> J

    J --> K[输入生产日期若需要、输入收货量]
    K --> L{收货量有效校验}
    L -->|无效| L1[提示: 数量录入不合规]
    L -->|实收+当前>计划| L2[提示: 实收总量大于计划量]
    L -->|有效且 实际+当前<=计划| M[新增收货记录 状态=初始]
    M --> N[更新入库单状态=收货中]
    N --> O[更新明细 实际入库量+=收货量]
    O --> P[调用库存规则 W001 库存变更]
    P --> Q[更新 inventory + inventory_log]
    Q --> R{策略 自动关闭入库单=是?}
    R -->|是| R1[全量收货执行关闭入库单逻辑]
    R -->|否| S[回到 扫描商品 页面]
    R1 --> S
    S --> End([继续收货或关闭])
```

## 4. UC03 取消收货

```mermaid
flowchart TB
    Start([收货员在录入页]) --> A[点击 收货记录]
    A --> B[进入收货记录页 默认 初始 页签]
    B --> C[点击 取消收货]
    C --> D{收货记录状态=初始?}
    D -->|否| D1[提示: 不是初始状态无法取消]
    D -->|是| E[更新收货记录状态=已取消]
    E --> F[更新明细 实际入库量-=收货量]
    F --> G[调用库存规则 W002 库存变更]
    G --> H{该入库单下是否全部收货记录=已取消?}
    H -->|是| H1[更新入库单状态=初始]
    H -->|否| I([结束])
    H1 --> I
```

---

## 5. UC04 关闭入库单

```mermaid
flowchart TB
    Start([收货员]) --> A{操作入口}
    A -->|移动端| A1[入库明细页 点击 关闭入库单]
    A -->|PC| A2[入库单列表 点击 关闭]
    A1 --> B[系统查验入库单状态]
    A2 --> B
    B --> C{状态 in 初始/收货中?}
    C -->|否| C1[提示: 入库单已关闭或取消无法关闭]
    C -->|是| D[弹出确认: 关闭后无法收货请确认]
    D --> E[收货员确认]
    E --> F[更新入库单状态=已关闭]
    F --> G{入口}
    G -->|移动端| G1[进入 查询入库单 页面]
    G -->|PC| G2([结束])
    G1 --> End([结束])
```

## 6. 全局校验（引用关系）

以下校验在文档「全局说明」中定义，在收货录入等流程中被引用：

```mermaid
flowchart LR
    subgraph 全局校验
        V1[商品有效校验\n仓库+货主+商品→sku_product]
        V2[商品数量有效校验\n称重/非称重规则]
        V3[状态机更新校验\n当前>=待更新则不更新]
        V4[任务作业权限校验\ntask_group→user_profile→profile→profile_detail]
        V5[储位有效校验\nlocation 启用=开启]
    end
```

## 7. 入库单状态与主流程关系

```mermaid
stateDiagram-v2
    [*] --> 初始: 生成入库单
    初始 --> 收货中: 首笔收货录入
    收货中 --> 收货中: 继续收货
    收货中 --> 已关闭: 关闭入库单
    初始 --> 已关闭: 关闭入库单
    收货中 --> 初始: 取消收货且全部记录已取消
```
