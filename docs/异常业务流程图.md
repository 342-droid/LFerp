# 出库异常管理业务流程图

## 1. 文档目标与范围

基于以下需求文档进行分析输出：
- `需求说明-异常.md`
- `表设计-异常.md`

覆盖范围：
- 异常触发建单（UC01）
- 异常受理（UC02）
- 确认有货（UC03）

## 2. 业务主流程图（端到端）

```mermaid
flowchart TD
    A["拣货、分拣、复核发生缺货或货损"] --> B["标记出库单异常"]
    B --> C["创建异常单头 obd_exception"]
    C --> D["创建异常单明细 obd_exception_detail"]
    D --> E["状态置为初始，处理完成置为否"]
    E --> F["用户查询并进入异常详情"]
    F --> G{"状态是否为初始"}
    G -- 否 --> G1["禁止受理"]
    G -- 是 --> H["执行受理"]
    H --> I["状态更新为已受理并记录操作日志"]
    I --> J{"异常类型是否为缺货或货损"}
    J -- 否 --> J1["其他类型待扩展"]
    J -- 是 --> K{"异常来源"}

    K -- 拣货 --> K1["执行 W004 释放占用库存"]
    K1 --> K2["执行 W008 移入异常位库存"]
    K -- 分拣 --> K3["执行 W008 将分拣来源库存移入异常位"]
    K -- 复核 --> K4["执行 W008 将复核来源库存移入异常位"]

    K2 --> L["显示确认有货与确认缺货按钮"]
    K3 --> L
    K4 --> L

    L --> M{"用户确认结果"}
    M -- 确认有货 --> N["状态更新为确认有货，处理完成为是"]
    M -- 确认缺货 --> O["状态更新为确认缺货，处理完成为是"]

    N --> N1["回写作业量"]
    N1 --> N2["执行 W009 从异常位回流业务储位"]
    N2 --> P{"同一出库单异常是否全部处理完成"}

    O --> O1["按来源扣减作业量"]
    O1 --> O2["执行 W010 创建并执行库存调整单"]
    O2 --> P

    P -- 是 --> Q["清除对应出库异常标志"]
    P -- 否 --> R["保持出库单异常标志"]
    Q --> S["流程结束"]
    R --> S
```

## 3. 分来源处理流程图（受理后）

```mermaid
flowchart LR
    A[已受理异常单] --> B{异常来源}
    B -->|拣货| C1[W004释放占用 -> W008入异常位]
    B -->|分拣| C2[W008分拣来源库存入异常位]
    B -->|复核| C3[W008复核来源库存入异常位]
    C1 --> D[等待人工确认]
    C2 --> D
    C3 --> D
    D --> E{确认有货/确认缺货}
    E -->|有货| F[W009回流至原业务储位并恢复生产]
    E -->|缺货| G[W010库存调整扣减并不可恢复]
```

## 4. 状态机与按钮控制建议

```mermaid
stateDiagram-v2
    [*] --> 初始
    初始 --> 已受理: 点击受理
    已受理 --> 确认有货: 点击确认有货
    已受理 --> 确认缺货: 点击确认缺货
    确认有货 --> [*]
    确认缺货 --> [*]
```

建议的前端按钮可见性/可点击性：
- `初始`：仅`受理`可点击，`确认有货/确认缺货`置灰
- `已受理`：`确认有货/确认缺货`可点击，`受理`置灰
- `确认有货`或`确认缺货`：全部操作按钮隐藏/禁用（终态）
