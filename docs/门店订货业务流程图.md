# 门店订货业务流程图

本文档基于 PRD《门店订货》与当前页面原型（`purchase_store_demand_summary.html`、`purchase_demand_summary.html`）整理，聚焦“门店订货汇总 → 门店订货单 → WMS出库联动”。

## 1. 全链路总览

```mermaid
flowchart TD
    A[C端消费者下单并支付成功] --> B[沉淀消费者订单明细]
    B --> C[采购进入 门店订货汇总]
    C --> D[按查询条件筛选并查看汇总]
    D --> E{是否截单生成门店订货单}
    E -- 否 --> D
    E -- 是 --> F[按门店维度汇总生成门店订货单]
    F --> G[回写订货单ID/单号到消费者订单明细]
    G --> H[通知WMS创建仓配出库单<br/>业务类型=配送出库]
    H --> I{配送仓库 = 收货仓库?}
    I -- 是 --> M[进入门店订货单履约状态流转]
    I -- 否 --> K[匹配路由规则并执行替代流程]
    K --> L[首节点仓库生成调拨出库单<br/>业务类型=仓库调拨]
    L --> M
```

## 2. UC01 门店订货汇总流程（截单并生成）

```mermaid
flowchart TD
    A[开始] --> B[进入 门店订货-门店订货汇总]
    B --> C[默认商品页签<br/>可切换 商品/门店/规格]
    C --> D[设置查询条件并点击查询]
    D --> E[展示汇总列表]
    E --> F{点击商品编码?}
    F -- 是 --> G[进入汇总详情]
    G --> H{修改实际订货量?}
    H -- 是 --> I[按修改后数量参与生成]
    H -- 否 --> J[按销售数量默认值参与生成]
    I --> K[返回汇总页]
    J --> K
    F -- 否 --> K
    K --> L{点击 截单并生成门店订货单?}
    L -- 否 --> D
    L -- 是 --> M[按门店维度生成门店订货单<br/>来源=门店订货汇总生成, 状态=初始]
    M --> N[回写消费者订单明细关联关系]
    N --> O[通知WMS先生成仓配出库单<br/>业务类型=配送出库]
    O --> P{配送仓库=收货仓库?}
    P -- 是 --> S[结束]
    P -- 否 --> R[按路由生成调拨出库]
    R --> S
```

## 3. UC02 门店订货单流程（新增/编辑/取消）

```mermaid
flowchart TD
    A[开始] --> B[进入 门店订货-门店订货单]
    B --> C[查询并浏览列表/详情]
    C --> D{执行动作}

    D -- 新增 --> E[点击 新增门店订货单]
    E --> F[填写主单与明细]
    F --> G{必填校验通过?}
    G -- 否 --> F
    G -- 是 --> H[保存成功<br/>来源=手动新增, 状态=初始]
    H --> I[通知WMS创建或同步出库]

    D -- 编辑 --> J[点击 编辑]
    J --> K{单据状态=初始?}
    K -- 否 --> X[拦截: 仅初始状态可编辑]
    K -- 是 --> L[仅可编辑允许字段]
    L --> M[保存并通知WMS同步变更]

    D -- 取消 --> N[点击 取消]
    N --> O{单据状态=初始?}
    O -- 否 --> Y[拦截: 仅初始状态可取消]
    O -- 是 --> P[门店订货单状态=已取消]
    P --> Q[通知WMS出库单状态同步]
    Q --> R{是否存在调拨链路?}
    R -- 否 --> Z[完成取消]
    R -- 是 --> S[按路由匹配调拨出库单]
    S --> T[回写取消数量到出库/波次明细]
    T --> U{订货批次是否全部取消?}
    U -- 是 --> V[对应WMS出库单状态=已取消]
    U -- 否 --> W[保留部分有效明细]
    V --> Z
    W --> Z

    I --> AA[结束]
    M --> AA
    X --> AA
    Y --> AA
    Z --> AA
```

## 4. 门店订货单状态流转

```mermaid
stateDiagram-v2
    [*] --> 初始
    初始 --> 待发货: WMS出库单集波
    待发货 --> 部分发货: WMS部分发货
    部分发货 --> 发货完成: WMS全量发货
    发货完成 --> 部分收货: 门店部分收货
    部分收货 --> 已完成: 门店全量收货
    初始 --> 已取消: 全量冲销/手动取消
    待发货 --> 已取消: 全量冲销
    部分发货 --> 已取消: 全量冲销
    发货完成 --> 已取消: 全量冲销
    部分收货 --> 已取消: 全量冲销
```

## 5. 关键规则摘录

- 数据来源：仅取 C 端下单并支付成功、且订单状态在`待发货/退款中`、且未关联门店订货单的订单明细。
- 汇总维度：页面支持`商品/门店/规格`查看，但生成门店订货单按门店维度汇总。
- 收货仓库规则：优先按路由规则匹配首个配送节点；匹配不到时收货仓库等于配送仓库。
- 订货单可编辑/可取消前提：仅`初始`状态允许。
- 取消联动：取消门店订货单时需同步变更 WMS 出库单与波次单已取消量，必要时将对应出库单置为`已取消`。
